# PURPOSE

Define executable mutation containment law for bounded MVP restoration.

---

# MVP MUTATION AUTHORITY

Authorization is corridor-scoped, not repository-scoped.

Mutation outside the currently active restoration corridor is forbidden.

Only bounded restoration mutations are authorized.

Mutation must:
- preserve runtime continuity
- preserve rollback capability
- preserve canonical authority boundaries
- preserve single-corridor restoration sequencing

---

# FORBIDDEN MVP MUTATIONS

- parallel runtime introduction
- duplicate executable authority creation
- temporary fallback runtime systems
- secondary participation-state derivation paths
- topology-wide normalization
- opportunistic adjacent cleanup
- historical reconstruction mutation
- cross-corridor mutation escalation
- implicit capability authority creation

---

# EXECUTION LAW

Executable mutation must preserve rollback-safe restoration boundaries.

Every bounded mutation must leave the runtime capable of returning to the last known stable continuity state.

If runtime continuity destabilizes during MVP restoration:

1. freeze further mutation
2. restore last known stable continuity
3. reclassify failure
4. resume bounded restoration only after continuity verification

Continuity restoration takes precedence over mutation progression.

---

# RESTORATION LAW

Bounded correction takes precedence over architectural perfection.

Continuity restoration takes precedence over historical parity.

Contracts override implementation residue.

No executable drift independently authorizes topology mutation.

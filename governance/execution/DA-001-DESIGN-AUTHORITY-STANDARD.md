Purpose

Define the constitutional governance for Founder-approved visual and
interaction intent.

Design Authority governs WHAT the Founder intends users to experience
visually.

Design Authority SHALL NOT:

• implement UI
• generate assets
• modify repository code
• redefine content
• redefine experience acceptance

Responsibilities

✓ Visual identity

✓ Branding

✓ Layout intent

✓ Information hierarchy

✓ Navigation intent

✓ Interaction intent

✓ Design consistency

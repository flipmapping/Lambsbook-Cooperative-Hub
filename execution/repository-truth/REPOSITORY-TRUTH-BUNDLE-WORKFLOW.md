# Repository Truth Bundle Workflow

Status

CANONICAL

Repository

Lambsbook-Open-Brain

Authority Stream

EOS

Type

Execution Infrastructure

--------------------------------------------------

Purpose

Define the canonical workflow for generating a Repository Truth Bundle (RTB).

--------------------------------------------------

Workflow

1.

Repository Bootstrap

- inspect repository;
- establish Repository Truth;
- certify Repository Truth.

2.

Repository Truth Bundle Generation

- identify bounded mutation corridor;
- identify implementation entry points;
- identify dependent runtime surfaces;
- collect required execution evidence;
- assemble Repository Truth Bundle.

3.

Builder

- consume certified Claude Instruction Brief;
- consume certified Repository Truth Bundle;
- materialize Claude Package.

4.

Claude

- consume Claude Package;
- perform bounded repository mutation.

5.

Founder

- verify implementation;
- certify work package.

--------------------------------------------------

Responsibility Boundary

Repository Bootstrap owns Repository Truth.

RTB Generation owns Execution Context.

Builder owns deterministic package materialization.

Claude owns bounded repository mutation.

Founder owns verification and certification.

--------------------------------------------------

Expected Outcome

Every repository mutation package SHALL be generated from:

- certified Repository Truth;
- certified Repository Truth Bundle;
- certified Claude Instruction Brief.

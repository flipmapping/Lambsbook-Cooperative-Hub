8. Fail-Fast Rule

Execution SHALL terminate immediately upon any repository-backed blocker
that prevents completion within the authorized mutation boundary.

Examples include:

✓ Required repository assets unavailable.
✓ Founder-approved CTBC media unavailable.
✓ Repository Reality differs from the certified baseline.
✓ Runtime contract violation.
✓ Build failure introduced by authorized mutations.
✓ Mutation required outside the authorized mutation boundary.
✓ New Founder authority superseding this Implementation Authority.

Upon termination, execution SHALL provide:

• Repository-backed evidence of the blocker.
• Affected mutation boundary.
• Impact on the implementation objective.
• Corrective authority required before execution may resume.

Execution SHALL NOT:

• Expand the mutation boundary.
• Introduce additional governance artifacts as compensation.
• Modify Builder, Repository Truth, runtime contracts, APIs, or constitutional authorities.


9. Stop Condition

Execution under GE-RMP-005 SHALL terminate when one of the following
conditions is satisfied.

Successful Completion

✓ All objectives defined by this Implementation Authority have been
  implemented.
✓ All required evidence defined by the Evidence Standard has been
  produced.
✓ Founder Acceptance Evidence has been obtained.
✓ Repository mutations are confined to the authorized mutation boundary.

Repository-backed Blocker

✓ A repository-backed blocker is encountered and execution terminates
  under the Fail-Fast Rule.

Superseding Authority

✓ The Founder authorizes a new Implementation Authority that supersedes
  GE-RMP-005.

Repository Reality Change

✓ Repository Reality objectively changes such that the certified
  Repository Truth Baseline is no longer valid.

Upon any Stop Condition, GE-RMP-005 SHALL be closed. Further repository
mutation SHALL require a new Founder-authorized Implementation Authority.
